## 🤝 团队协作

### 任务分配原则

1. **技能匹配**: 根据团队成员技能分配任务
2. **工作量平衡**: 确保工作量合理分配
3. **依赖关系**: 考虑任务间的依赖关系
4. **优先级**: 优先处理高优先级任务

### 讨论机制

- **任务开始前**: 讨论技术方案和实现细节
- **任务进行中**: 定期同步进度和遇到的问题
- **任务完成后**: 代码 review 和验收确认

### 沟通渠道

- **技术讨论**: GitHub Issues / Pull Requests
- **进度同步**: 定期站会
